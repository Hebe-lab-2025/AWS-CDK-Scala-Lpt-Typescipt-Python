# 🧠「一句话秒选卡」——DB 扩展三件套（必背）

> 目标：**看到关键词 → 不解释 → 直接选**

---

## 🟦 Multi-AZ（高可用，不提性能）

**一句话秒选卡：**

> **题干只提 HA / failover / availability → Multi-AZ**

**看到这些词就锁死：**

* high availability
* automatic failover
* no single point of failure
* disaster recovery（同 Region）

**千万别选：**

* ❌ Read Replica
* ❌ Cache
* ❌ Aurora（除非题干明确 performance）

---

## 🟩 Read Replica（读扩展）

**一句话秒选卡：**

> **读多 / 报表 / analytics / read-heavy → Read Replica**

**看到这些词就锁死：**

* read-heavy workload
* reporting / analytics
* scale reads
* offload read traffic

**千万别选：**

* ❌ Multi-AZ（它不提速）
* ❌ Cache（除非提 sub-ms）

---

## 🟨 Cache（性能 / 延迟）

**一句话秒选卡：**

> **low latency / sub-ms / offload DB → Cache**

**看到这些词就锁死：**

* sub-millisecond latency
* reduce database load
* hot data
* read latency spike

**千万别选：**

* ❌ Read Replica（不够快）
* ❌ Multi-AZ（完全不对路）

---

## ⚡ 3 行终极对照（考场版）

```
HA → Multi-AZ
Read scale → Read Replica
Latency → Cache
```

> **如果题干只够你选一个 → 永远按这三行来**
